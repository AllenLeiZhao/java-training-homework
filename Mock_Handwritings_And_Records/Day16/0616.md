## 2. If we override hashCode but not equals?
 
If you override hashCode but not equals, hash-related data structures like HashMap, HashSet, and ConcurrentHashMap will have issues during hash collision or data update. When two objects hash to the same bucket, the data structure uses equals to determine if they are the same key. Since equals is not overridden, it falls back to the default reference comparison from Object, so two objects with identical content are treated as different keys. This causes incorrect behavior — you may end up with duplicate entries in a HashSet, or a HashMap get may fail to find a value even though the key looks identical. The rule is: always override equals and hashCode together.
 
## 3. How do you write a REST API in Spring Boot?
 
To build a REST API in Spring Boot, I follow the three-tier architecture. I define the entity with JPA annotations, create a repository extending JpaRepository, add a service for business logic, and create a controller with @RestController and @RequestMapping for the base URL. I use @GetMapping for read, @PostMapping for create, @PutMapping for update, and @DeleteMapping for delete. Input comes from @PathVariable for URL path values, @RequestParam for query strings, @RequestHeader for headers, and @RequestBody to deserialize the request body. Every method returns a ResponseEntity with the data and the appropriate status code. I also add input validation using @Valid on the request body combined with constraint annotations like @NotBlank and @Email on the DTO fields — if validation fails Spring throws a MethodArgumentNotValidException. For exception handling I use @RestControllerAdvice with @ExceptionHandler methods to catch exceptions globally and return clean error responses with the correct status code, so I do not need try-catch blocks in every controller. Two rules I always follow: the URL is a noun never a verb, and every response returns both payload and status code.
 
## 4. What is thread state?
 
A Java thread has six states. New means it has been created but not started. Runnable means start() has been called and it is waiting for the CPU scheduler. Running means it is currently executing. Blocked means it is waiting to acquire a lock held by another thread. Waiting means it is waiting indefinitely after calling wait() or join(). Timed Waiting means it is waiting for a specific duration after calling sleep(), wait with a timeout, or join with a timeout.Terminated means the run() method has finished.
 
## 5. What is ThreadLocal?
 
ThreadLocal provides each thread with its own independent copy of a variable so threads do not interfere with each other. The most common use case is storing the current logged-in user in a web application — you set the user at the start of the request and any code in that thread can access it without passing it as a parameter. The critical rule is to always call remove() at the end of the request, because web applications use thread pools and threads are reused. If you do not clean up, the next request on the same thread will see the previous user's data, causing a data leak.
 
## 6. What is CORS?
 
CORS stands for Cross-Origin Resource Sharing. It is a browser security mechanism that enforces the same-origin policy, which blocks requests from a web page to a different domain, protocol, or port. For example if Angular runs on localhost 3000 and the Spring Boot backend runs on localhost 8080, the browser blocks the request by default because the ports differ. In Spring Boot I handle it by adding @CrossOrigin on specific controllers, or by configuring a global CORS policy using WebMvcConfigurer to specify allowed origins and HTTP methods. On the Angular side, developers must also handle the difference between synchronous and asynchronous modes when making HTTP calls, using async and await for non-blocking requests.
 
## 7. Map vs filter?
 
Both are Stream API operations. Filter takes a predicate and keeps only elements that match the condition, reducing the number of elements but keeping the same type. Map takes a function and transforms each element, keeping the same number of elements but potentially changing the type. For example, filter on a list of employees keeps only those older than 30, while map transforms each Employee into just their name, producing a stream of Strings. They are commonly chained: filter first to narrow the data, then map to transform it, then collect to produce the final result.
 
## 8. Thread lifecycle?
 
The thread lifecycle starts at New when the object is created. Calling start() moves it to Runnable. The scheduler picks it up and it enters Running. If it tries to acquire a locked resource it goes to Blocked. Calling wait() or join() moves it to Waiting. Calling sleep() or wait with a time parameter moves it to Timed Waiting. When run() finishes it enters Terminated. The key distinction is Blocked waits for a lock, while Waiting waits for an explicit notify from another thread.
 
## 9. By-name vs by-type injection?
 
Spring injects by type first. If exactly one bean matches the required type it is injected directly. If multiple beans of the same type exist, Spring falls back to by-name using the variable name. If that also fails, Spring throws a NoUniqueBeanDefinitionException at startup. To resolve this, use @Primary to mark one bean as the global default, or use @Qualifier at the injection point to specify the exact bean name. A practical example is having multiple datasource beans — you use @Qualifier to inject the correct one for each use case. 
 
## 10. How to send a request from Angular to backend?
 
In Angular you use the HttpClient module to send HTTP requests. You import HttpClientModule into the app module, inject HttpClient into a service, and call get(), post(), put(), or delete() with the backend URL. These return Observables which the component subscribes to for the response data. For POST and PUT you pass the request body as the second argument. You also need to handle CORS on the backend side to allow requests from the Angular origin. Angular HTTP calls are asynchronous by default using Observables, and you can also use async and await with promises as an alternative for a more synchronous-style approach.
 
## 11. What is pattern matching?
 
Pattern matching was introduced in Java 16 and simplifies type checking by combining instanceof and casting into one expression. Before pattern matching, you had to write instanceof to check the type and then manually cast it to use it. With pattern matching you write instanceof followed by the type and a variable name — if the check passes the variable is automatically bound to the cast value with no explicit cast needed. Java 21 extended pattern matching to switch statements, where each case can directly match on a type like integer, string, or long and manipulate the value without manual casting.
 
## 12. Spring Boot Actuator?
 
Spring Boot Actuator is a production-ready monitoring module that exposes HTTP endpoints for health checks, metrics, and application state. You add the spring-boot-starter-actuator dependency and configure which endpoints to expose in application.properties. Key endpoints are /health, /metrics, /loggers, /beans, and /caches. You can also create custom endpoints using @Endpoint with @ReadOperation, @WriteOperation, or @DeleteOperation. Micrometer is used to collect metrics, which are stored in Prometheus as a time-series database, and then visualized in Grafana as dashboards. In production only expose the endpoints you need, because endpoints like /env can leak sensitive configuration.
