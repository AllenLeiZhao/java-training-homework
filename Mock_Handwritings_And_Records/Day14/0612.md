## 1	introduce what is Spring Framework
Spring is a lightweight Java framework for building enterprise Java applications. Its two core principles are Inversion of Control and Aspect-Oriented Programming. IOC means the Spring container takes over the responsibility of creating and managing objects and injecting dependencies automatically. AOP allows you to separate cross-cutting concerns like logging and security from business logic. Spring evolved from the original Spring Framework into Spring MVC for web development, and finally into Spring Boot which adds auto-configuration to eliminate boilerplate configuration — which is the version I've been using in my project.

## 2	spring boot version you used
In my project I used Spring Boot 3, which is built on Spring Framework 6 and requires a minimum of Java 17.

Spring Boot 3 was a major release because it migrated all package names from javax to Jakarta — that's the most significant breaking change when upgrading from Spring Boot 2. For example, javax.persistence.Entity becomes jakarta.persistence.Entity. It also added native image support through GraalVM and better built-in observability with Micrometer.

## 3	how do you define profile
A profile in Spring Boot allows you to define environment-specific configurations for environments like development, testing, UAT, and production, without changing your code. There are two ways to define a profile. First, create separate configuration files for each environment — either in properties or YAML format — named application dash the profile name, like application-dev or application-prod. Second, use the @Profile annotation on a bean or configuration class so that bean is only loaded when a specific profile is active. To activate a profile, you can set it in application.properties, pass it as a JVM argument, set it as a system environment variable, or use @ActiveProfiles on a test class.

## 4	what discovery service impl you used before
I used Netflix Eureka through Spring Cloud as our service discovery mechanism.
Eureka acts as a centralized registry — a board where all microservices register themselves and monitor each other's health. Each service registers by name instead of IP address, because in a cloud environment IP addresses change constantly. Each client sends periodic heartbeats to the Eureka Server to show it's alive. If the server stops receiving heartbeats, it deregisters that service automatically. On the server side I add @EnableEurekaServer to the main class. On each client I add @EnableEurekaClient and configure the service name and Eureka Server URL in application.properties.

## 5	what is aop
AOP stands for Aspect-Oriented Programming. It separates cross-cutting concerns like logging and security from business logic so you don't duplicate that logic throughout the application.
Spring AOP works through proxies — Spring wraps your bean in a proxy that intercepts method calls. The key is defining both the "where" and the "when": the Pointcut defines where to intercept, for example all methods in the service layer, and the Advice defines what to do at that point. There are five advice types: Before, After, AfterReturning, AfterThrowing, and Around. AOP cannot apply to private methods because the proxy only intercepts public method calls from outside the class. Also, calling a method with this dot inside the same class bypasses the proxy entirely.

## 6	how to write spring boot to call from frontend to backend and save data to database
The frontend sends an HTTP POST request with data in the request body. The controller receives it, passes it to the service for business logic, and the service calls the repository to persist it to the database.
The flow follows the three-tier architecture consistently: the Controller layer is annotated with @RestController and uses @PostMapping to expose the endpoint. It uses @RequestBody to deserialize the incoming JSON into a Java object, and @Valid to trigger input validation. The Service layer handles business logic and validation rules. The Repository layer extends JpaRepository, and Spring Data JPA uses ORM to generate the SQL and save the data. The controller wraps the response in a ResponseEntity with the saved data and a 201 Created status code.

## 7	Describe Spring MVC
Spring MVC is a web framework built on the Model-View-Controller pattern where every request flows through a central DispatcherServlet.
Model is the data, View is what gets returned to the client, and Controller handles the request and coordinates between the two. The DispatcherServlet acts as the front controller — it receives all requests and uses a handler mapping to route them to the correct controller method based on the HTTP method and URL combination. In a REST API with @RestController, the View step is skipped and the return value is serialized directly to JSON. Spring Boot registers and configures DispatcherServlet automatically, replacing the need for manual configuration files like web.xml.

## 8	how do you validate input data in spring boot
I add constraint annotations to the fields in the DTO or Entity class, and add @Valid to the controller method parameter to trigger automatic validation.
Constraint annotations include @NotNull to reject null values, @NotBlank for empty strings, @Email for email format, @Min and @Max for number ranges, @Size for length limits, and @Pattern for custom regular expressions. When @Valid is placed on the @RequestBody parameter, Spring automatically validates the incoming data before it reaches the business logic. If any constraint is violated, Spring throws a MethodArgumentNotValidException, which my @RestControllerAdvice catches and returns a 400 Bad Request with a descriptive error message.

## 9	Spring boot actuator
Spring Actuator is a built-in monitoring tool that exposes HTTP endpoints to check the health, metrics, and internal state of a running application without stopping it.
The required dependency is spring-boot-starter-actuator. Key endpoints include: slash actuator slash health for application and dependency health, slash actuator slash metrics for memory usage and request counts, slash actuator slash loggers to change log levels at runtime without restarting, slash actuator slash beans to see all registered Spring beans, and slash actuator slash caches to view active caches. In production, best practice is to only expose the endpoints you actually need in application.properties, because endpoints like slash env can leak sensitive heap memory information to external users. I use Actuator together with Prometheus for data collection and Grafana for visualization.

## 10	how does spring mvc work
Every request goes through DispatcherServlet, which uses HandlerMapping to find the correct controller method, delegates the request, and sends back the response.
The full seven-step flow from David Lu's diagram: one, request comes in from the web browser. Two, DispatcherServlet consults HandlerMapping to find the correct controller method based on HTTP method and URL. Three, DispatcherServlet routes the request to the Controller. Four, Controller processes it through Service and Repository layers and returns ModelAndView. Five, DispatcherServlet passes the view name to ViewResolver. Six, ViewResolver returns the View. Seven, the rendered response is sent back to the client. In a REST API with @RestController, steps five and six are skipped — the return value is serialized directly to JSON using HttpMessageConverter.

## 11	what is controller how you use controller how you implement controller
A Controller is the top layer of the three-tier architecture, responsible for exposing RESTful endpoints and routing incoming HTTP requests to the correct service logic.
To implement it, I annotate the class with @RestController to register it as a Spring bean that returns JSON responses. I use @RequestMapping at the class level to set the base URL path. Then I define methods for each CRUD operation: @GetMapping for read, @PostMapping for create, @PutMapping for update, and @DeleteMapping for delete. I use @PathVariable to extract values from the URL path, @RequestParam for query string parameters, and @RequestBody to deserialize the request body. I wrap all responses in ResponseEntity to control both the response body and the HTTP status code. I also add global exception handling with @RestControllerAdvice to significantly improve code robustness.

## 12	what is webflux? Have you used it in your project
Spring WebFlux is a non-blocking reactive web framework that uses the Reactor library for asynchronous programming, designed for high-concurrency scenarios.
Traditional Spring MVC uses a thread-per-request model in Tomcat — each request holds a thread until it completes, so threads sit idle waiting for responses. WebFlux is non-blocking — threads don't wait, they handle other tasks and resume when data arrives. WebFlux uses Mono for a single value and Flux for a stream of values. One important note from David Lu: with Java 21, virtual threads combined with modernized Tomcat significantly improve thread performance, which reduces the necessity for reactive programming in many cases. Some projects still maintain reactive styles due to existing investment, but for new projects virtual threads are often simpler. I haven't used WebFlux in production, but I understand its trade-offs.

## 13	how you connect the database in springboot
I add the Spring Data JPA dependency and the database driver to pom.xml, then configure the connection properties in application.properties.
Spring Boot auto-configures the datasource and entity manager automatically based on those properties. On the code side, I annotate the entity class with @Entity and @Id to map it to a database table. I create a repository interface extending JpaRepository, which uses ORM to generate all standard CRUD operations automatically without writing SQL. The repository is injected into the service layer. For handling return values from the repository, I use Optional to wrap the result and prevent null pointer exceptions — for example Optional ofNullable prevents a NullPointerException when a record is not found.

## 14	how do you handle global exception in spring boot
I use @RestControllerAdvice with @ExceptionHandler methods to catch specific exception types globally and return clean error responses to the client.
I create one class annotated with @RestControllerAdvice as the central exception handling location. Inside it, each method is annotated with @ExceptionHandler for a specific exception type — for example a ResourceNotFoundException returns 404, a MethodArgumentNotValidException returns 400, and unexpected exceptions return 500. Each handler returns a ResponseEntity with the status code and a descriptive error message. AOP can also be used for exception handling through @AfterThrowing advice, which is useful for logging exceptions centrally. The two work together: AOP handles logging the exception details, @RestControllerAdvice handles returning the clean response to the client.

## 15	Spring boot annotation
Spring Boot annotations fall into several categories: bean declaration, dependency injection, web layer, validation, and configuration.
Bean declaration: @Component is generic, @Service for business layer, @Repository for data layer with database exception translation, @Controller and @RestController for web layer, @Bean inside @Configuration for third-party classes. @SpringBootApplication combines @Configuration, @EnableAutoConfiguration, and @ComponentScan. Web layer: @RequestMapping, @GetMapping, @PostMapping, @PutMapping, @DeleteMapping, @PathVariable, @RequestParam, @RequestBody, @ResponseBody. Validation: @Valid to trigger validation, @NotNull, @NotBlank, @Email, @Size, @Min, @Max, @Pattern. Configuration: @Value to inject values from properties files into Java code without an application reboot — useful for dynamic settings like cache expiration times. Scope: @Scope to specify singleton, prototype, request, or session.

## 16	how Spring IOC work , all annotations and injection and bean types
Spring IOC means the ApplicationContext container takes over the responsibility of creating and managing objects. When the application starts, it reads annotations, scans for beans, creates bean definitions, instantiates them, and performs dependency injection.
Bean registration annotations: @Component, @Service, @Repository, @Controller, @RestController are placed on classes and auto-scanned. @Bean is placed on a method inside @Configuration for third-party classes whose source code you can't modify. Injection types: constructor injection is recommended because it is explicit, guarantees valid state, and simplifies testing — this is the fail-fast approach. Field injection is convenient but hides dependencies and is hard to test. Setter injection is rarely used. If circular dependencies occur, use @Lazy to break the cycle. Bean scopes: Singleton (default, one instance per IOC container), Prototype (new instance every time, Spring does not manage destruction), Request (one per HTTP request), Session (one per user session), Application (one per ServletContext).

## 17	how many ways to inject bean in spring and which one we use most
There are three ways: constructor injection, setter injection, and field injection. Constructor injection is the most recommended and most commonly used.
Constructor injection makes all dependencies explicit and required at object creation time, which prevents NullPointerException at runtime — this is the fail-fast behavior David Lu emphasized. It also makes unit testing much easier because you can pass mock objects directly through the constructor. Field injection with @Autowired is convenient but private fields can't be easily set from outside the class, making tests fragile and hiding issues until runtime. Setter injection makes dependencies optional which can lead to incomplete object states, so it's rarely used today.

## 18	by name vs by type
Spring tries by-type first. If exactly one bean matches the required type it gets injected directly. If multiple beans of the same type exist, Spring falls back to by-name using the variable name to match the bean name.
If neither resolves the ambiguity, Spring throws an error at startup. To resolve conflicts: @Primary marks one bean as the global default choice — Spring automatically prefers it when there's ambiguity. @Qualifier is used at the injection point to specify exactly which bean you want by name, giving precise control at each injection site. A practical example from David Lu: when migrating from Oracle to PostgreSQL, you might have two datasource beans. Use @Qualifier to inject the correct one where needed, and @Primary to set the default.

## 19	why constructor injection
Constructor injection is recommended because it makes dependencies explicit, guarantees the object is always in a valid state, and makes unit testing much easier — this is the fail-fast approach.
All required dependencies must be provided at the time the object is created, so the object can never exist in an incomplete state. This prevents NullPointerException at runtime. For testing, you can pass mock objects directly through the constructor without needing the Spring container. Field injection is the opposite — private fields can't be easily set from outside, so you need reflection to inject mocks, which makes tests fragile and can hide issues until runtime. Spring itself recommends constructor injection for all mandatory dependencies.

## 20	what java version can we use with spring boot 3
Spring Boot 3 requires a minimum of Java 17. It also supports Java 21, which introduces Virtual Threads that significantly improve concurrent performance.
Java 17 is required because Spring Boot 3 is based on Jakarta EE 10 and Spring Framework 6. The most important change when upgrading from Spring Boot 2 is the javax to Jakarta package rename — all imports need to be updated. With Java 21, Virtual Threads allow Tomcat to handle far more concurrent requests with minimal overhead, which in many cases reduces the need for reactive programming with WebFlux. David Lu noted that virtual threads combined with modernized Tomcat make thread performance much better without the debugging complexity of reactive code.

## 21	what is dispatcherservlet
DispatcherServlet is the front controller in Spring MVC. It receives all incoming HTTP requests and uses a handler mapping to route them to the correct controller method.
When a request comes in, DispatcherServlet consults the handler mapping, which identifies the correct controller method based on the HTTP method and URL combination. It routes the request to that controller, gets the result, and in a traditional MVC application passes the view name to ViewResolver to render the HTML response. In a REST API with @RestController, the view step is skipped and the return value is serialized directly to JSON. Spring Boot registers and configures DispatcherServlet automatically, replacing the need for manual configuration files like web.xml that older Spring projects required.
