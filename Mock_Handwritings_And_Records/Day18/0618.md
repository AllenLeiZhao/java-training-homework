## 1. What Spring Boot version did you use?
 
I have used both Spring Boot 2 and 3. My most recent project uses Spring Boot 3. The biggest difference between the two is that Spring Boot 3 requires Java 17 as the minimum version, and it migrated all packages from javax to jakarta. On top of that, Spring Boot 3 added GraalVM native image support which compiles the application into a native executable with much faster startup time, and starting from version 3.2 it supports Java 21 virtual threads for better concurrency.
 
## 2. How to write a REST API in Spring Boot?
 
To write a REST API in Spring Boot, I follow the three-tier architecture with Controller, Service, and Repository layers. Starting from the bottom, I define the Entity with @Entity and @Id to map it to a database table, and create a Repository extending JpaRepository for automatic CRUD operations. In the Service layer I write the business logic. In the Controller I annotate the class with @RestController and @RequestMapping for the base URL, then use @GetMapping, @PostMapping, @PutMapping, and @DeleteMapping for each operation. For input I use @PathVariable for URL path values, @RequestParam for query strings, @RequestHeader for headers, and @RequestBody with @Valid to deserialize and validate the request body. Every method returns a ResponseEntity with the appropriate HTTP status code. I also add @RestControllerAdvice with @ExceptionHandler for global exception handling so validation failures return clean error responses. Two design rules I always follow: the URL is a noun never a verb, and every response includes both a payload and the correct status code.
 
## 3. Difference between recursion and iteration?
 
Both repeat work, but recursion solves a problem by calling itself on a smaller sub-problem while iteration uses a loop. The key practical difference is memory — recursion adds a stack frame for every call, so deep inputs can cause StackOverflowError and consume more memory. Iteration uses constant stack space and is generally faster. Recursion tends to be more elegant for naturally recursive structures like trees and graphs. Iteration is safer and more performant for simple linear repetition. The rule of thumb is: use recursion when the problem is naturally recursive, use iteration for deep or simple linear inputs.
 
## 4. What is a fair lock?
 
A fair lock guarantees that threads acquire the lock in the order they requested it — first in first out — which prevents thread starvation. In Java you create one with ReentrantLock constructed with the boolean true. The trade-off is performance: a fair lock is slower than a non-fair lock because it must maintain a waiting queue and check order every time. The default ReentrantLock is non-fair, which allows threads to barge in and grab the lock, making it faster overall but potentially starving some threads. Use a fair lock only when ordering matters and starvation is not acceptable.
 
## 5. What is a sealed class?
 
A sealed class introduced in Java 17 restricts which classes are allowed to extend or implement it. You declare it with the sealed keyword and use the permits clause to explicitly list the allowed subclasses. Each permitted subclass must be declared as either final (cannot be extended further), sealed (can be extended but with its own permits), or non-sealed (open for extension without restriction). The main benefit is exhaustive pattern matching — when you use a sealed class in a switch expression, the compiler knows all possible subtypes and can warn you if you miss a case. It is commonly used to model fixed sets of types like payment methods or shapes where you want to control the type hierarchy.
 
## 6. Introduce what is Spring Framework?
 
Spring is a lightweight Java framework for building enterprise applications with Inversion of Control and Aspect-Oriented Programming as its two core principles. IOC means the Spring container takes over the responsibility of creating and managing objects — instead of manually wiring dependencies, you declare them and Spring injects them automatically. AOP allows you to separate cross-cutting concerns like logging and security from business logic. Spring evolved from the original Spring Framework into Spring MVC for web development, and finally into Spring Boot which adds auto-configuration to eliminate boilerplate setup. The core container is ApplicationContext, which scans annotations, creates bean definitions, instantiates beans, and performs dependency injection at startup. In my project I used Spring Boot 3, which is the most current version of this evolution.
 
## 7. What is Consumer?
 
Consumer is a functional interface in the java.util.function package introduced in Java 8. It represents an operation that takes a single input argument and returns no result — it consumes the value without producing anything. The abstract method is accept(T t). It is commonly used with forEach on streams or collections, where you pass a Consumer lambda to define what to do with each element, like printing it or saving it. You can also chain multiple Consumers using the andThen method, which executes the first Consumer and then the second on the same input. Consumer is one of the core functional interfaces alongside Function which takes input and returns output, Predicate which takes input and returns boolean, and Supplier which takes no input and returns a value.
 
## 8. How does Spring IOC work, all annotations, injection types, and bean types?
 
Spring IOC means the ApplicationContext container takes over creating and managing objects. At startup it reads annotations, performs component scanning to find all beans, creates bean definitions, instantiates the beans, and performs dependency injection automatically. If circular dependencies occur, use @Lazy to defer creation of one bean.
 
For bean registration: @SpringBootApplication combines @EnableAutoConfiguration, @ComponentScan, and @Configuration. Under scanning: @RestController for the web layer, @Service for business logic, @Repository for data access with database exception translation, @Component for generic beans, and @Bean inside a @Configuration class for third-party classes you cannot annotate directly.
 
For dependency injection: @Autowired triggers injection. Spring injects by type first — if exactly one match is found it injects directly. If multiple beans of the same type exist it falls back to by name using the variable name. If that also fails it throws NoUniqueBeanDefinitionException. Resolve with @Primary for a global default or @Qualifier at the injection point for precise control. Three injection types: constructor injection is recommended because it is fail-fast, prevents NullPointerException, and makes unit testing easy. Field injection is convenient but hard to test. Setter injection makes dependencies optional and is rarely used.
 
For bean scope: Singleton is the default — one instance per IOC container, suitable for stateless components. Prototype creates a new instance every time requested and Spring does not manage its destruction. Request creates one per HTTP request. Session creates one per user session. Application creates one per ServletContext.
