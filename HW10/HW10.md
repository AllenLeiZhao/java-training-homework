## 1. what is AOP
AOP stands for Aspect Oriented Programming. It is used to separate cross-cutting concerns like logging, security, and transaction management from business logic. Instead of writing repetitive code in every method, you define these concerns once in an Aspect class and AOP automatically intercepts the target methods and applies the logic. For example, instead of adding logging to every service method, you define it once and Spring automatically injects it before or after all specified methods run. Common AOP annotations include @Aspect, @Pointcut, @Before, @After, and @Around.

## 2. what is JointPoint and Aspect in AOP
Aspect is a class that contains the cross-cutting logic you want to apply to multiple methods, such as logging or security checks. JoinPoint represents the specific method execution point that is being intercepted, providing information about the method name, arguments, and target object at runtime.

## 3. what is Aspect, joinpoint, pointcut, advice, target
Aspect is a separate class that contains cross-cutting logic like logging or security that you want to apply to multiple methods. Pointcut defines where and when the aspect should be applied, using expressions to match specific packages, classes, or methods. JoinPoint represents the specific method currently being intercepted at runtime. Advice is the actual action to execute at the joinpoint, such as @Before, @After, or @Around. Target is the object instance that owns the method being intercepted by the aspect.

## 4. ApplicationContext vs BeanFactory
BeanFactory is the root interface of Spring IoC container, providing basic Bean management like getBean(). ApplicationContext is a sub-interface of BeanFactory that extends it with additional enterprise features such as AOP support, event publishing, and eager bean initialization at startup. In practice, ApplicationContext is always used instead of BeanFactory directly. Spring Boot uses AnnotationConfigWebApplicationContext as the default implementation.

## 5. How is Spring MVC Working Flow 
In Spring MVC, all incoming HTTP requests first go through the DispatcherServlet, which acts as the front controller. It uses HandlerMapping to find the correct Controller and method to handle the request. The Controller calls the Service layer for business logic, which then communicates with the Repository layer for database operations. The result is returned back to the Controller, then DispatcherServlet uses HttpMessageConverter to serialize the response object into JSON and sends it back to the client.

## 6. how does @Autowire work
@Autowired tells Spring to automatically inject a matching Bean from the IoC container. Spring first searches by type, and if multiple beans of the same type exist, it falls back to matching by name. If still ambiguous, @Primary or @Qualifier must be used to specify which bean to inject. Constructor injection is recommended over field injection because it supports final fields and makes testing easier.

## 7. @Autowire and @Qualifer
@Autowired tells Spring to automatically inject a matching Bean from the IoC container. When there are multiple beans of the same type, Spring cannot decide which one to inject and will throw an error. @Qualifier solves this by explicitly specifying which bean to inject by name. It is different from @Primary which sets a default bean — @Qualifier gives you precise control over which specific implementation to use.

## 8. 2 use cases for @PostConstruct and @PreDestroy
@PostConstruct is executed after the Bean is fully initialized and all dependencies are injected. Common use cases include preloading cache from database and establishing connections at startup. @PreDestroy is executed just before the Bean is destroyed when the container shuts down. Common use cases include closing database connections and releasing file resources to prevent resource leaks.
